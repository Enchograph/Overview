# Overview 固定接力提示词

新开一个空对话时，直接发送下面这句：

```text
请作为 Overview 项目的接力开发代理继续工作。不要先问我背景，先完整阅读仓库里的 docs/PROJECT_RULES.md、docs/PROJECT_STATUS.md、docs/ROADMAP.md、docs/HANDOFF.md 和当前阶段的 docs/todos/*.md，然后检查 git 状态与最近提交，接着直接执行“下一步唯一推荐动作”。默认持续自主开发直到项目完成；每完成一个功能块都要编译、测试、修复问题、更新状态文档并提交 git。遇到安装工具链、提权、删除/回退、部署发布或其他高风险命令时，必须先向我说明并等待我确认，不能自行执行。除非遇到仓库内无法推断且会造成高风险返工的决策，否则不要停下来问我。
```

## 这句提示词的作用

- 明确告诉新代理这是一个“接力开发”场景
- 强制新代理先读仓库内规则与状态，而不是依赖聊天上下文
- 强制新代理直接继续开发，不停留在总结或规划
- 强制新代理把编译、测试、修复和 Git 提交纳入同一工作链
- 强制新代理在高风险命令前停下并等待用户审批

## 使用方式

1. 开新对话
2. 把上面的整句提示词发给 Codex
3. Codex 应自动读取仓库状态并继续开发

## 预期结果

只要仓库内的规则、状态、TODO 和交接记录持续维护，这句固定提示词就足够让后续代理无缝续跑。
