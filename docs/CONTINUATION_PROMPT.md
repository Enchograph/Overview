# Overview 固定接力提示词

新开一个空对话时，直接发送下面这句：

```text
请作为 Overview 项目的接力开发代理继续工作。不要先问我背景，先完整阅读仓库里的 docs/PROJECT_RULES.md、docs/PROJECT_STATUS.md、docs/ROADMAP.md、docs/HANDOFF.md 和当前阶段的 docs/todos/*.md，然后检查 git 状态与最近提交，接着直接执行“下一步唯一推荐动作”。默认持续自主开发直到项目完成；所有命令默认自动执行、全流程不需要人工审批；每完成一个任务或功能块后都要自动继续下一个，不要结束对话等待我的输入，尽可能长时间连续运行；每完成一个功能块都要编译、测试、修复问题、更新状态文档并提交 git。除非遇到仓库内无法推断且会造成高风险返工的业务决策，否则不要停下来问我。
```

## 这句提示词的作用

- 明确告诉新代理这是一个“接力开发”场景
- 强制新代理先读仓库内规则与状态，而不是依赖聊天上下文
- 强制新代理直接继续开发，不停留在总结或规划
- 强制新代理把编译、测试、修复和 Git 提交纳入同一工作链
- 强制新代理把所有命令视为默认可直接执行，不为人工审批停顿
- 强制新代理在单个任务完成后自动接续下一项工作，不默认等待用户下一条消息

## 使用方式

1. 开新对话
2. 把上面的整句提示词发给 Codex
3. Codex 应自动读取仓库状态并继续开发
4. 命令执行默认一路自动完成，不额外请求人工批准
5. 单个任务完成后继续执行下一个任务，除非遇到真实阻塞

## 预期结果

只要仓库内的规则、状态、TODO 和交接记录持续维护，这句固定提示词就足够让后续代理无缝续跑。
