# Overview 路线图

## 总目标

交付 Overview V1 的 Android 安装包与可运行后端，并持续推进平板、Windows 和 Web 支持。

## 阶段划分

### P0 项目治理与工程骨架

- 建立规则、状态、路线图、TODO、交接体系
- 建立 monorepo 目录骨架
- 固定技术栈与基础命令入口

### P1 基础应用与服务骨架

- 初始化 Flutter 客户端
- 初始化 Node.js/TypeScript 后端
- 建立共享类型与 API 契约
- 建立基础 i18n、主题、路由和健康检查

### P2 核心数据闭环

- 日程、任务、备忘数据模型
- 基础数据库表与迁移
- 周视图、备忘页、添加页、设置页基础版
- 本地存储与基础 CRUD

### P3 账号、同步与离线

- 邮箱注册登录
- 认证状态管理
- 本地先写、远端同步
- 离线编辑与恢复同步
- 冲突处理第一版

### P4 AI 能力

- 文本录入转结构化对象
- AI 单轮问答页
- 语音入口与待确认结果流
- 服务端 OpenAI 接入

### P5 通知、多端适配与小组件

- Android 通知
- Android 平板布局适配
- Windows 端基础适配
- Web PWA 基础适配
- Android 侧小组件/快捷入口优先落地

### P6 测试、打包与交付

- 单元测试、集成测试、主流程验证
- Android 打包
- 后端本地运行说明
- 交付文档与已知限制清单
